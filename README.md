# codex-mode

CLI for switching Codex between ChatGPT subscription mode and any
OpenAI-compatible ChatCompletions provider (DeepSeek, etc.) via
[@codeproxy/cli](https://github.com/codeproxy-ai/cli).

`codex-mode` manages Codex config and starts the proxy. The proxy is
vendored as a git submodule under `vendor/codeproxy/`.

## Install

```bash
git clone --recurse-submodules https://github.com/annajcy/codex-mode.git
cd codex-mode
./install.sh
```

Or step by step:

```bash
git clone --recurse-submodules https://github.com/annajcy/codex-mode.git
cd codex-mode

# Python CLI
uv tool install -e .

# Proxy from submodule
cd vendor/codeproxy
npm install --ignore-scripts
npm run build
npm link
```

## Usage

Add DeepSeek:

```bash
codex-mode add deepseek --token your-api-key
```

Add a custom OpenAI-compatible ChatCompletions provider:

```bash
codex-mode add custom \
  --base-url https://api.example.com \
  --token your-api-key \
  --model provider-model
```

Switch Codex to the local proxy provider:

```bash
codex-mode api deepseek
```

Run Codex:

```bash
codex-mode run "Say OK"
```

Return to ChatGPT subscription mode:

```bash
codex-mode sub
```

## Files

| Path | Purpose |
|------|---------|
| `~/.codex/config.toml` | Codex config (backed up before each switch) |
| `~/.codex/codex-mode-profiles.json` | API keys (mode `600`) |
| `~/.codeproxy/config.json` | Proxy upstream config (generated by `codex-mode api`) |

## Update

```bash
git pull --recurse-submodules
cd vendor/codeproxy && npm run build && npm link && cd -
```
